# Session Handoff

## File
- `SESSION_CONTEXT.md` in project root.

## Required Sections
- Project scope and key file paths.
- What changed in this session (behavior + UI).
- Current constraints/known caveats.
- Next likely steps.

## Style
- Keep concise and accurate.
- Prefer bullet points.
- Include only actionable context for the next session.
