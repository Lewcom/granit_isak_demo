# DB Transition

Робоча папка для ідей, рішень і узгоджених правил переходу на нову оптимізовану структуру бази.

## Що зберігаємо тут

- цільову структуру таблиці;
- бізнес-правила (як рахується ціна, які поля обов'язкові);
- mapping зі старої структури в нову;
- відкриті питання;
- прийняті рішення (з датою).

## Поточна робоча ідея структури

- `component_code`
- `component`
- `pricing_type` (`single|double`, технічне поле для правильної ціни)
- `colors`
- `param1`, `value1`
- `param2`, `value2`
- `param3`, `value3`
- `option`
- `option_value` (раніше обговорювали як `option_price_delta`)
- `notes`
- `price`

## Принцип заповнення

- `price` = базова ціна рядка без додаткової опції;
- `option` + `option_value` описують додаткову опцію/коригування;
- підсумкова ціна рядка з опцією: `price + option_value`;
- `pricing_type` не є бізнес-параметром компонента, це технічна колонка фільтрації прайсу;
- якщо параметрів менше трьох, зайві `param/value` лишаються порожніми.
