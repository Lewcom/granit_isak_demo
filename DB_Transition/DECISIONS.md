# Decisions Log

## 2026-03-06

- Створено папку `DB_Transition` як єдине місце для підготовки переходу.
- Зафіксовано модель з парами `paramN/valueN` для підтримки компонентів з кількома параметрами.
- Погоджено, що `option_price_delta` у попередній термінології відповідає полю `option_value`.
- На цьому етапі `mtype` і `category` не включаються до нової цільової таблиці.
- Додаємо стабільний ідентифікатор `component_code` (обов'язкове поле).
- Формула ціни з опцією: `final_price = price + option_value`.
- Розбиваємо змішані параметри:
  - `Бардюр верхній`: `тип` -> `ширина` + `форма`.
  - `Хвильки 4шт`: `розмір` -> `товщина` + `діапазон`.
- Формат `params_dictionary.json`: `param_name -> components[] -> allowed_values[]`.
- `single|double` переносимо з параметрів у технічну колонку `pricing_type`.
- `variant` більше не зберігається в `param3/value3`.
